# HTTP
## 2026-07-20 Mon
### 국제적인 컨텐츠를 지원하기 위한 HTTP 스펙
- HTTP 메시지는 컨텐츠라면 기반 언어를 불문하고 미디어처럼 실어나를 수 있으며, 이 관점에서 HTTP 엔티티 본문은 그저 비트의 연속에 불과하다.
- 반면, 국제 컨텐츠를 지원하기 위해 서버는 클라이언트가 알 수 있도록 각 문자와 언어를 명시할 수 있다.
  - 이를 통해 클라이언트는 문자를 구성하는 비트들을 올바른 방법으로 처리하여 사용자에게 컨텐츠를 제공할 수 있다.
- 이렇듯 서버가 클라이언트에게 전달하는 문자와 언어 정보는 `charset`과 Content-Language 헤더에 명시된다.
- 동시에 클라이언트는 서버에게 어떠한 언어를 이해할 수 있고, 처리할 수 있는지 명시할 필요가 있다.
  - 이 때 사용되는 헤더는 Accept-Charset과 Accept-Language이며, 각각 어떤 인코딩 알고리즘과 어떤 언어를 이해할 수 있는지 명시한다.

### 차셋이란?
- HTTP 차셋(`charset`)은 임의의 컨텐츠 비트들을 특정한 문자 체계의 글자들로 바꾸는 방법을 의미한다.
- 각 차셋은 임의의 차셋 태그를 명시하며, 이러한 태그들은 각각 비트를 글자로 변경하거나 그 역의 작업을 수행하는 알고리즘에 대한 명명에 해당한다.
  - 이러한 차셋 태그의 목록은 표준화되어 IANA에 의해 관리된다.
- 예를 들어 `Content-Type: text/html; charset=iso-8859-6`과 같은 헤더는 컨텐츠 수신자에게 다음과 같은 정보를 제공한다.
  1. 컨텐츠는 HTML 파일이다.
  2. 컨텐츠를 구성하는 비트들은 아랍 문자집합 디코딩 기법인 `iso-8859-6`에 의해 글자로 디코딩되어야 한다.
- 상술한 `iso-8859-6`의 경우 8비트 값을 몇몇 특수문자를 포함하는 라틴 문자 또는 아랍 문자로 매핑하는 알고리즘에 해당한다.
  - 반면, UTF-8과 같은 몇몇 문자 인코딩 방식은 글자 당 비트 수가 일정하지 않기에 처리 과정이 더 복잡할 수 있다.
  - 그러나 이러한 인코딩 방식은 중국어나 일본어와 같이 절대적인 글자 수가 다른 언어에 비해 많은 문자체계를 지원할 수 있다는 장점 역시 존재한다.
