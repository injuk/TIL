# HTTP
## 2026-07-20 Mon
### 국제적인 컨텐츠를 지원하기 위한 HTTP 스펙
- HTTP 메시지는 컨텐츠라면 기반 언어를 불문하고 미디어처럼 실어나를 수 있으며, 이 관점에서 HTTP 엔티티 본문은 그저 비트의 연속에 불과하다.
- 반면, 국제 컨텐츠를 지원하기 위해 서버는 클라이언트가 알 수 있도록 각 문자와 언어를 명시할 수 있다.
  - 이를 통해 클라이언트는 문자를 구성하는 비트들을 올바른 방법으로 처리하여 사용자에게 컨텐츠를 제공할 수 있다.
- 이렇듯 서버가 클라이언트에게 전달하는 문자와 언어 정보는 `charset`과 Content-Language 헤더에 명시된다.
- 동시에 클라이언트는 서버에게 어떠한 언어를 이해할 수 있고, 처리할 수 있는지 명시할 필요가 있다.
  - 이 때 사용되는 헤더는 Accept-Charset과 Accept-Language이며, 각각 어떤 인코딩 알고리즘과 어떤 언어를 이해할 수 있는지 명시한다.

## 2026-07-21 Tue
### 차셋이란?
- HTTP 차셋(`charset`)은 임의의 컨텐츠 비트들을 특정한 문자 체계의 글자들로 바꾸는 방법을 의미한다.
- 각 차셋은 임의의 차셋 태그를 명시하며, 이러한 태그들은 각각 비트를 글자로 변경하거나 그 역의 작업을 수행하는 알고리즘에 대한 명명에 해당한다.
  - 이러한 차셋 태그의 목록은 표준화되어 IANA에 의해 관리된다.
- 예를 들어 `Content-Type: text/html; charset=iso-8859-6`과 같은 헤더는 컨텐츠 수신자에게 다음과 같은 정보를 제공한다.
  1. 컨텐츠는 HTML 파일이다.
  2. 컨텐츠를 구성하는 비트들은 아랍 문자집합 디코딩 기법인 `iso-8859-6`에 의해 글자로 디코딩되어야 한다.
- 상술한 `iso-8859-6`의 경우 8비트 값을 몇몇 특수문자를 포함하는 라틴 문자 또는 아랍 문자로 매핑하는 알고리즘에 해당한다.
  - 반면, UTF-8과 같은 몇몇 문자 인코딩 방식은 글자 당 비트 수가 일정하지 않기에 처리 과정이 더 복잡할 수 있다.
  - 그러나 이러한 인코딩 방식은 중국어나 일본어와 같이 절대적인 글자 수가 다른 언어에 비해 많은 문자체계를 지원할 수 있다는 장점 역시 존재한다.

## 2026-07-22 Wed
### 적절한 차셋 사용의 중요성
- 문자를 사용하기 위해서는 우선 비트들을 문자로 변환하는 디코딩 알고리즘을 지정하고, 이를 적용하는 표준화된 방법이 필요할 수 있다.
- 국제화된 문자 시스템의 목표는 시각적인 표현 방식인 표현에서 글자들 그 자체인 의미를 분리하는 것인 반면, HTTP는 문자 데이터와 차셋 전송에만 관심을 갖는다.
- 때문에 클라이언트가 잘못된 `charset` 매개변수를 사용한다면 클라이언트는 서버로부터 받은 컨텐츠를 잘못 디코딩하므로 깨진 데이터를 보여주게 된다.
  - 예를 들어, 본문이 서유럽 문자 코드 인코딩인 `iso-8859-1`로 인코딩되었지만 클라이언트가 다른 인코딩을 사용할 경우 의도와는 다른 문자가 출력된다.

## 2026-07-23 Thu
### MIME 차셋이란?
- 특정한 문자 인코딩과, 특정하게 코딩된 문자집합의 결합은 MIME 차셋이라는 용어로 지칭한다.
  - 이러한 MIME 차셋은 IANA에 의해 등록되어 관리되며, HTTP의 경우 표준화된 MIME 차셋 태그를 Content-Type과 Accept-Charset 헤더에 사용한다.
- 여러 문서와 브라우저에 의해 사용될 수 있는 MIME 차셋 인코딩 구조의 예시 중 가장 유명한 두 종류는 다음과 같다.
  1. us-ascii: ASCII라고도 불리는 유명한 문자 인코딩으로, 여러 국제 변형이 있기에 정확한 식별을 위해 `us-` 접두사를 붙인다.
  2. utf-8: 전 세계 문자들에 대한 보편적인 문자 집합인 유니코드를 표현하기 위한 가변길이 문자 인코딩 구조이다.

## 2026-07-24 Fri
### <META> 태그와 문자 집합 추론 
- 웹 서버는 클라이언트에게 MIME 차셋 태그를 `charset` 매개변수화 함께 다음과 같이 전송할 수 있다.
```text
Content-Type: text/html; charset=iso-2022-jp
```
- 만약 이러한 문자집합이 명시되지 않았다면, 수신자는 문서의 `<META>` 등과 같은 태그로부터 문자 집합을 추측하려 시도한다. 
  - 그러나 이러한 정보조차 주어지지 않았다면 언어와 인코딩에 대한 일반적인 패턴을 찾아내기 위해 텍스트를 읽어들여 문자 인코딩을 추측하는 방식으로 동작한다.
  - 반면, 클라이언트가 어떠한 정보로부터도 문자 인코딩을 추측할 수 없다면 서유럽 언어를 위한 인코딩 방식인 iso-8859-1로 가정하게 된다.
